# Migrations legadas

Este diretório preserva, sem alteração de conteúdo, as migrations geradas durante a fase Lovable/desenvolvimento da base anterior.

Esses arquivos:

- são somente inventário técnico e histórico;
- não fazem parte da sequência oficial da nova base limpa;
- não devem ser aplicados, copiados ou usados como backfill;
- podem ser consultados apenas para recuperar aprendizados sobre módulos e riscos do legado.

A sequência oficial da reconstrução controlada em Next.js + Supabase limpo começa em `supabase/migrations/`.
